## 搬瓦工速度测试的三种方式

**搬瓦工（BandwagonHost）**是加拿大 IT7 Networks 旗下的 VPS 商家，在美国、日本、香港、荷兰等地都有机房，不同机房对国内用户的网络质量差异非常明显。在买套餐之前，建议先用下面三种方式摸清楚哪个机房和你本地网络最搭。

### 方式一：本地 ping 测试（最快，1 分钟搞定）

这是最直接的办法。搬瓦工官方为每个机房提供了测试 IP，拿 ping 工具打一遍，延迟和丢包率就出来了。

**Windows 操作步骤**：

1. 按 `Win + R`，输入 `cmd`，回车打开命令行
2. 输入命令：`ping 对应机房IP`（IP 见下方表格）
3. 看返回的 `time=` 数值，这就是延迟（毫秒）
4. 观察是否有 `Request timeout`，有的话说明存在丢包

**Mac / Linux 操作步骤**：

1. 打开终端（Terminal）
2. 输入：`ping 对应机房IP`
3. 按 `Ctrl+C` 停止，结果里同样看延迟和丢包率

---

### 方式二：全国延迟测试（更全面）

本地 ping 只代表你自己所在地的情况。用 `ping.chinaz.com` 可以从全国几十个城市同时测试对应 IP，一眼看出这个机房在不同地区、不同运营商下的延迟分布。

操作很简单：打开站长工具，把机房 IP 填进去，点测试，等一两分钟就能看到结果。

---

### 方式三：SpeedTest + LookingGlass 演示站测速

搬瓦工为部分机房搭建了演示站点，可以直接跑 SpeedTest 看实际下载速度，也可以用 LookingGlass 工具测试路由跟踪，确认是否走了 CN2 GIA 线路。

对于想确认"是否真的走 CN2 GIA"的用户，这步很有用，直接看 traceroute 里的 AS 号即可。

## 各机房国内速度参考：哪个延迟最低

延迟不是唯一指标，但延迟差太多体感会非常明显。下面是对国内用户来说，各机房综合表现的排序参考：

**香港 CN2 GIA（HKHK_8）**
物理距离最近，广东方向延迟低至 9ms，华南用户基本是物理极限。三网全程 CN2 GIA，晚高峰抗拥堵能力也是所有机房里最强的。缺点就是贵，$89.99/月起，适合对延迟要求很高、预算也够的用户。

**洛杉矶 DC6 CN2 GIA-E / DC9 CN2 GIA**
这两个机房是大多数用户的主力选择。国内延迟通常在 140-180ms 区间，对日常建站、挂脚本基本够用，电信和联通走 CN2 GIA，稳定性比普通线路好不少。

**日本大阪软银（JPOS_1）/ 日本东京（DC39）**
对华南、华东用户来说，延迟和洛杉矶差不多，有时候甚至更低。软银线路移动体验相对好，日本机房的 ping 值通常在 100ms 出头。东京 DC39 走 CMI 三网回程，移动用户反馈不错。

**荷兰 EUNL_9**
三网接入联通 AS9929，对联通用户友好，移动和电信就一般了。如果主力是联通宽带，可以测一下。

**洛杉矶 DC2 / KVM 系入门机房**
普通国际线路，延迟基本在 200ms 以上，用来学习 Linux、跑脚本没问题，但对访问速度敏感的场景就不太合适。

说实话，**CN2 GIA 和普通线路的差距，晚高峰最明显**。白天可能差别不大，晚上八九点国内网络拥堵，普通线路可能 300ms+ 还丢包，CN2 GIA 还是 150ms 左右。

## 搬瓦工中文网速度测试：常见问题解答

**Q：ping 延迟低就代表速度快吗？**

不完全是。ping 测的是往返延迟，实际下载速度还受带宽、路由、晚高峰拥塞等因素影响。延迟低是个好的开始，但最好再用 SpeedTest 测一下实际吞吐量，或者下载一个测速文件感受一下实际速率。

**Q：测试 IP 和我买的 VPS IP 是一样的吗？**

不是。测试 IP 是搬瓦工官方挂在对应机房的专用测速节点，你买了 VPS 之后会有自己独立的 IP 地址，但线路和机房是一样的，所以测试 IP 的延迟可以作为参考。

**Q：买了之后可以换机房吗？**

可以，CN2 GIA-E 套餐在 KiwiVM 后台支持免费一键切换 12+ 个机房。KVM 套餐可以切换的机房少一些（8 个普通机房）。香港套餐不支持切换到其他机房。

**Q：搬瓦工有没有优惠码？**

**Q：速度测试显示延迟很高怎么办？**

先确认测的是对应机房的 IP，别搞错了。其次，延迟高有时候是运营商和目标机房之间路由不佳导致的，换一个机房测测看。搬瓦工 CN2 GIA 线路设计上就是为了解决拥塞问题，如果普通机房延迟高，升级到 CN2 GIA-E 套餐通常有明显改善。
